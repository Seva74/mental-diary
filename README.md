# Mental Diary

Mental Diary — рабочий beta-прототип дневника состояния для фазового задания. В корне лежит документация и служебные файлы, а само приложение находится в `mental-diary-app`.

Текущая сборка уже включает гостевые сессии, пользовательский контур данных, локальную ML-оценку, AI-рекомендации через OpenAI/OpenRouter/fallback, безопасный support-модуль и прототипные форум с блогом.

## Что сейчас работает

- Ежедневные записи с настроением, энергией, сном, стрессом, заметками и тегами.
- Офлайн-черновики в вебе: если API недоступен, запись сохраняется локально и потом отправляется вручную.
- Гостевые сессии с bearer-токеном. Веб-клиент сам создаёт сессию и хранит токен в `localStorage`.
- Пользовательский дашборд, анализ, рекомендации и история предсказаний.
- Локальная ML-оценка по истории записей и текстовым маркерам.
- AI-совет через OpenAI, OpenRouter или локальный fallback, в зависимости от конфигурации.
- Безопасные support-действия вместо медицинских направлений.
- Прототипные форум и блог.
- Светлая и тёмная тема, адаптивная вёрстка и навигация по `Alt+1` - `Alt+6`.

## Как устроены анализ и AI

`MentalStateModel` оценивает историю записей локальной небольшой нейросетью. Затем `computeAnalysis` объединяет этот результат с rule-based порогами и считает риск, тренд, нагрузку стресса, защитный ресурс и итоговую сводку.

Рекомендации собираются в несколько слоёв:

1. Правила по текущей аналитике.
2. Короткая AI-сводка по анализу и последним записям.
3. Локальная рекомендация на следующую запись дневника.

AI-адаптер выбирается через `AI_PROVIDER`:

- `openai` использует `OPENAI_API_KEY` и `OPENAI_MODEL`.
- `openrouter` использует `OPENROUTER_API_KEY` и `OPENROUTER_MODEL`.
- `fallback` остаётся полностью локальным.

Промпт специально зажат по формату: ответ должен быть по-русски, коротким, практичным, без цитирования сырых записей и метрик, без диагностического тона и с мягкой рекомендацией обратиться за живой поддержкой, если в тексте есть признаки самоповреждения. Если внешний провайдер недоступен, вернул пустой ответ или ошибку, код автоматически переходит на локальную рекомендацию.

Support-подсистема работает отдельно от AI. `SUPPORT_PROVIDER=local` оставляет встроенный self-help-план. `SUPPORT_PROVIDER=remote` может сходить во внешний API и всё равно откатится на локальные действия при таймауте или ошибке.

Снимки предсказаний сохраняются при пересчёте аналитики и не дублируются, если подпись состояния не изменилась. Поэтому `/api/predictions` может содержать меньше записей, чем число сохранённых действий.

## Сессии и офлайн-режим

- Гостевая сессия живёт 30 дней.
- Токен хранится в `localStorage` под ключом `mental-diary-session-token`.
- При ответе `401` клиент очищает токен и создаёт новую гостевую сессию.
- Офлайн-черновики хранятся в `localStorage` под ключом `mental-diary-offline-entries-v2`.
- Синхронизация черновиков идёт вручную и последовательно; список очищается только после успешной отправки всех записей.

## Структура проекта

- `mental-diary-app/apps/api` — Express API, storage, auth/session, анализ, рекомендации, support, форум/блог, метаданные системы и локальная ML-модель.
- `mental-diary-app/apps/web` — React/Vite UI, hooks, views, тема, офлайн-очередь и автоматическая работа с гостевой сессией.
- `mental-diary-app/apps/web/server.mjs` — статический сервер SPA с проксированием `/api` и `/health`.
- `mental-diary-app/docker-compose.yml` — PostgreSQL, API и web-сервис.
- `mental-diary-desktop.html` — дизайн-референс для текущего интерфейса.

## Запуск через Docker

```bash
cd mental-diary-app
docker compose up --build
```

После запуска:
- Web UI: http://localhost:8080
- API: http://localhost:3001
- Health: http://localhost:3001/health
- System meta: http://localhost:3001/api/system/meta

## Локальная разработка

```bash
cd mental-diary-app
npm install
npm run dev:api
npm run dev:web
```

## Сборка и тесты

```bash
cd mental-diary-app
npm run build
npm run test
```

`npm run test` сейчас запускает только backend-тесты. Web-тесты пока не заведены.

## Переменные окружения

Скопируйте `mental-diary-app/.env.example` в `mental-diary-app/.env`, если хотите переопределить дефолты.

| Переменная | Назначение |
| --- | --- |
| `PORT` | Порт API. |
| `DATABASE_URL` | Строка подключения к PostgreSQL. Если не задана, используется память. |
| `FRONTEND_ORIGIN` | Разрешённый origin для CORS web-клиента. Если не задан, CORS сейчас открыт для всех источников, поэтому для продакшена его нужно задать явно. |
| `AI_PROVIDER` | `openai`, `openrouter` или `fallback`. |
| `OPENAI_API_KEY` | Ключ OpenAI. |
| `OPENAI_MODEL` | Имя модели OpenAI. |
| `OPENROUTER_API_KEY` | Ключ OpenRouter. |
| `OPENROUTER_MODEL` | Имя модели OpenRouter. |
| `SUPPORT_PROVIDER` | `local` или `remote`. |
| `SUPPORT_API_URL` | Адрес внешнего support API. |
| `SUPPORT_TIMEOUT_MS` | Таймаут внешнего support API. |
| `VITE_API_BASE_URL` | Базовый URL API для веб-клиента. В Docker-сборке используется `/api`. |
| `VITE_API_TARGET` | Таргет Vite-прокси для `/api` и `/health` в локальной разработке. |
| `API_TARGET` | Таргет прокси для `apps/web/server.mjs` в production/Docker. |

## API

| Метод | Маршрут | Что делает |
| --- | --- | --- |
| `POST` | `/api/auth/guest-session` | Создаёт гостевую сессию и токен. |
| `GET` | `/api/auth/me` | Возвращает текущий контекст сессии. |
| `GET` | `/api/dashboard` | Возвращает полный пользовательский дашборд. |
| `GET` | `/api/entries` | Возвращает записи текущего пользователя. |
| `POST` | `/api/entries` | Сохраняет новую запись дневника. |
| `GET` | `/api/analysis` | Возвращает текущий анализ. |
| `GET` | `/api/recommendations` | Возвращает рекомендации, включая AI-сводку. |
| `GET` | `/api/predictions` | Возвращает историю предсказаний текущего пользователя. |
| `GET` | `/api/support` | Возвращает безопасные support-действия. |
| `GET` | `/api/forum/posts` | Возвращает форумные посты. |
| `POST` | `/api/forum/posts` | Создаёт форумный пост. |
| `GET` | `/api/blog/articles` | Возвращает статьи блога. |
| `GET` | `/api/system/meta` | Возвращает статус AI, support и ML-интеграций. |
| `GET` | `/health` | Возвращает статус сервиса и краткое состояние подсистем. |

`/health` сейчас возвращает объект вида `{ status, storageMode, aiProvider, mlProvider }`. `GET /api/system/meta` возвращает `storageMode`, а также вложенные блоки `ai`, `support` и `ml` с полями `provider`, `mode`, `configured`, `description` и `contract`.

`/api/support` возвращает массив `SupportAction` с полями `id`, `title`, `summary`, `action`, `reason` и `priority`. Публичными также остаются `GET /api/forum/posts`, `POST /api/forum/posts`, `GET /api/blog/articles`, `GET /api/system/meta` и `GET /health`.

Защищённые маршруты принимают `Authorization: Bearer <token>` или `X-Session-Token`. Веб-клиент отправляет bearer-токен автоматически после создания гостевой сессии.

## Ограничения

- Полноценные password-based аккаунты пока не реализованы, используется гостевой режим.
- AI и support могут откатываться на локальную логику, если внешние сервисы не настроены или недоступны.
- Форум и блог — прототипные модули; форумные посты не привязаны к пользователю, а блог отдаёт демо-статьи из seed-данных.
- Локальная ML-модель объяснимая и лёгкая, но это не медицинская диагностика.
- На старте в память и/или БД подгружаются demo-данные, чтобы дашборд не был пустым.
